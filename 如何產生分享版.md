# 故宮速記 — 分享 / 啟動 說明

## 日常自己看（React 版）

```powershell
cd web
npm run dev
```
- 你自己跑時，網址是 **http://localhost:5173/**（Vite 預設）。
- 看完關掉終端機即可；下次再 `npm run dev`。
- 不需要 build、不會碰到崩潰。

---

## 要分享給別人 → 用靜態版（推薦，零依賴）

根目錄的 **`故宮速記_分享版.zip`** 已經打包好，內含：
- `index.html`（資料全內嵌，單檔）
- `img/`（補光後的展品圖）

對方解壓後，**雙擊 `index.html`** 就能看，不用裝任何東西、不用網路、不用 server。
> 這份是靜態版（深色美術館風 + 篩選 + 精選 + 影片 + 深入查詢），功能完整，只少了 React 版的滑覽動效。

重新產生這個 zip（資料有更新時）：
```powershell
cd C:\Users\yu2_7\Downloads\故宮0625
python _build.py ; python _html.py          # 重生 index.html（資料更新才需要）
Compress-Archive -Path index.html,img -DestinationPath 故宮速記_分享版.zip -Force
```

## 進階：要產生 React 版的可分享 dist（需要切 Node 22）

目前 Node 24 在這台 `npm run build` 會原生崩潰（rollup 原生套件的已知問題，與專案碼無關）。
切到 Node 22 LTS 就正常：

1. 裝 **nvm-windows**（一次性）：到 https://github.com/coreybutler/nvm-windows/releases 下載 `nvm-setup.exe` 安裝。
2. 切換並重裝相依：
   ```powershell
   nvm install 22
   nvm use 22
   cd C:\Users\yu2_7\Downloads\故宮0625\web
   npm install        # 用 Node 22 重建原生套件
   npm run build      # 產出 web\dist\
   ```
3. `web\dist\` 即可分享（`base:'./'`，雙擊 `dist\index.html` 也能開）。
4. 要切回日常：`nvm use 24`。

> 不想搞 Node 切換就忽略這段 —— 自己看用 `npm run dev`，分享用上面的靜態 zip，兩條路都不需要 build。
